# `%s: %m`

## What it means

A passthrough shape: the whole message is `%s: %m` — a caller-supplied string followed by the OS error. It is used by client tools (`psql`, `pg_dump`, and friends) to report a system-call failure with a short context prefix. The real meaning depends entirely on the prefix and the `errno` text.

## When it happens

Emitted from many client-side sites whenever a libc call fails and the tool wants to prepend what it was doing. Because both parts are filled at runtime, the static text tells you only the shape, not the specific failure.

## How to fix

Read the rendered line: the part before the colon says what the tool was attempting, and the `%m` part (the `errno` text) says why it failed. Diagnose from those — for example `could not open file "x": Permission denied`. This page documents the passthrough pattern; the specific guidance lives with the concrete operation.

## Example

*Illustrative* — a rendered instance of the `%s: %m` shape.

```text
psql: error: connection to server failed: Connection refused
```
