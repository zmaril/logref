# `column "%s" can only be updated to DEFAULT`

## What it means

An `UPDATE` tried to set a `GENERATED ALWAYS AS IDENTITY` column to an explicit value. The placeholder is the column name. Such an identity column may be set only to `DEFAULT`, which lets the system assign the next value.

## When it happens

Running `UPDATE t SET id = <value>` where `id` is defined `GENERATED ALWAYS AS IDENTITY`, without overriding the system value.

## How to fix

Set the column to `DEFAULT` (or omit it) so the system controls the value, or if you must write an explicit value, use `OVERRIDING SYSTEM VALUE` on an `INSERT` — note that this override applies to inserts, not ordinary updates. Reconsider whether the identity column should be `GENERATED BY DEFAULT` if you need to set it directly.

## Example

*Illustrative* — updating a GENERATED ALWAYS identity to a value.

```sql
UPDATE t SET id = 100;
-- ERROR:  column "id" can only be updated to DEFAULT
```
