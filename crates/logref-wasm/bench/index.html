logref scanner throughput — specialized-matcher WASM vs hand-tuned TS

Same catalog, same rendered log lines, in the browser. Specialized WASM is the purpose-built printf matcher (anchored Aho-Corasick candidates + regex-free byte-loop verification) via scanBatchPacked(lines) — the whole result as ONE flat Int32Array, reconstructed JS-side by slicing each line by the returned byte spans (that decode cost is charged here). Trigram WASM is the PREVIOUS wasm path (trigram prefilter + regex-crate verify), same packed marshalling. Nested WASM is the specialized matcher behind serde-marshalled scanBatch. Hand-tuned TS is the production ScanIndex (trigram prefilter + V8-JIT RegExp, per-line scanLine).
